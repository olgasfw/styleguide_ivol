Overview

The iVolunteers style guide is a user interface component library designed to promote visual consistency, unify visual design and aid in any future development of the iVolunteers system. This working document should get updated, should any new components get added or the existing ones modified. The look and feel of each iVolunteers instance may be customized through the admin tool to reflect the client brand identity: logo, color scheme of select elements and footer navigation.

Color Pallet

Default Color

Neutral greys, used for text and backgrounds, blue, reserved for links, brand-specific accent green, used throughout the application for navigation, task performance and content highlight purposes, and alert red, used for error messaging and missing fields identification, are part of the iVolunteers template color scheme.

  • Subheader: 80 characters max

Navigation

Header Navigation

iVolunteers main navigation is sticky Bootstrap-based navigation that remains static on top of the screen as the page is being scrolled vertically. Then the resolution of the output devise is at or below 768px, the navigation collapses into a "burger" menu which will expand vertically in responce to being clicked click.

The "accent" color of the navigation links and the background color of the header may be customized to meet each client’s stylistic guidelines.

Standart Display

Display at or below 768px

Page Level Navigation

Page level navigation on the landing screen is used to "filter" betweeen Petitions and/or Surveys.

In addition, Page level navigation on the user account management screen allows to toggle thought various user accounts options.

Footer Navigation

The content of the footer navigation is client specific and is to be entered in .html format through the iVolunteers admininstrative interface.

Buttons

There are two type of buttons in iVolunteers: .btn-primary (conveys most emphasis) and .btn-default. The width of the button is controlled by the Grid, or .col-.. class that has been assigned to the <div> in which the button is placed. The styling of the buttons can be customized according to each client's needs.

Grid System

Grid Overview

iVolunteers is build based on Bootstrap's responsive, mobile first fluid grid system that scales up to 12 columns. The grid consists of predefined classes that allow for various layout options.

  • Rows are used to create horizontal groups of columns and are placed within a fixed-width .container or full-width .container-fluid for proper alignment and padding.
  • Content should be placed into columns .col-..-.., and columns can the be the only immediate children of rows.
  • Predefined grid classes, such as .row and .col-xs-4 are used for making grid layouts.
  • The spacing between the columns, or gutters, is controlled through bootstrap.css pre-defined padding
  • If more than 12 columns are placed within a single row, each group of extra columns will, as one unit, wrap onto a new line.
  • Grid classes apply to devices with screen widths greater than or equal to the breakpoint sizes, and override grid classes targeted at smaller devices: applying any .col-md-* class to an element will not only affect its styling on medium devices but also on large devices if a .col-lg-* class is not present.

Column Ordering

Order of built-in grid columns can be changed by adding .col-md-push-* and .col-md-pull-* modifier classes.

1st: .col-md-9 .col-md-push-3
2nd: .col-md-3 .col-md-pull-9

Element Visibility

Page elements (not limited to columns) can be hidden by adding .hidden-* modifier classes. Similarly, hidden columns can be made visible by adding .visible-* modifier class.
